VirtualBox

VirtualBox ist eine von der Firma Oracle entwickelte Virtualisierungslösung, die es dem Benutzer erlaubt, weitere Betriebssysteme (Gäste) unter einem laufenden System (Host) zu installieren und wie eine normale Anwendung zu nutzen. Als Hostsysteme werden Windows ab XP, Mac OS X, Linux (ab Kernel 2.4) und FreeBSD (ab 7.0), als Gastsysteme neben diesen zusätzlich noch Windows NT und 2000, OS/2, DOS-basierte Betriebssysteme, Linux (ab Kernel 2.2), L4, Solaris, NetWare sowie diverse BSD-Derivate unterstützt (siehe auch Liste der Gastsysteme ).

Seit dem 15. Januar 2007 steht VirtualBox in zwei Versionen zur Verfügung: Eine Open-Source-Edition (kurz: OSE), die unter der GPL v2 veröffentlicht wird, und eine PUEL-Variante (Personal Use and Evaluation License), die unter bestimmten Bedingungen kostenlos verwendet werden darf. Ab Version 4.0 wird VirtualBox erst durch Installation des Extension-Packs zur PUEL-Version (12/2010). Deren Lizenzbedingungen sind recht weit gefasst, so dass der Gebrauch am heimischen PC bedenkenlos möglich ist. Sollte man sich entscheiden, das Extensionspack einzusetzen, so muss unbedingt der Gleichklang der Versionsnummern beachtet werden.

Seit 2010 gehört VirtualBox, durch Übernahme von Sun Microsystems, zu Oracle. Neue Versionen, die Anpassungen, Fehlerbereinigungen und neue Eigenschaften beinhalten, werden in einem mehr oder minder gleichbleibenden monatlichen Turnus veröffentlicht.

Merkmale, die der Open-Source-Edition fehlen:

  • USB 2.0 & 3.0 - Unterstützung (seit Version 4.0 wird USB 1.1 in der OSE-Version unterstützt)

  • Remote Display Protocol (RDP) Server

  • USB über RDP

  • iSCSI-Unterstützung

innotek hatte angekündigt, dass die Funktionen der proprietären Version schrittweise in die Open-Source-Edition einfließen sollen. Vom neuen Eigentümer Oracle wurde dies bisher erst teilweise umgesetzt (Evaluation unter eindeutiger PUEL-Lizenz möglich).

Der Umgang wird in folgenden Unterartikeln gesondert behandelt:

  • Installation - Installation der beiden Versionen und die nötigen Schritte zur Nachbearbeitung

  • Erster Start und Benutzung - Bedienung und die verschiedenen Einstellungen

  • Netzwerk - die verschiedenen Möglichkeiten, wie das Gastsystem in ein Netzwerk eingebunden werden kann

  • Problembehebung - aufgetretene Probleme, für die bereits eine Lösung gefunden wurde

  • Dualboot-Windows virtualisieren - installiertes Windows (und andere Systeme) sowohl mit VirtualBox nutzen, als auch als Dualboot-System
