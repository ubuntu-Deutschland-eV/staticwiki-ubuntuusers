Open DC Hub

Das Open DC Hub Paket ist alles andere als gut gepflegt. An vielen Stellen trifft man auf Ungereimtheiten oder Bugs. Bis das Paket auf einen aktuellen Stand gebracht wurde, sollte man Open DC Hub mit Vorsicht genießen.

Open DC Hub ist "Hub" für das Filesharing-Protokoll Direct Connect. Das DirectConnect-Protokoll erlaubt P2P-Filesharing, also von Benutzer-zu-Benutzer. Dabei erfordert es einen zentralen "Server", den so genannten Hub. Die einzelnen Nutzer melden sich am Hub an, welcher sich dann um die Verwaltung der Suchanfragen kümmert und dafür sorgt, dass sich die Nutzer überhaupt finden können.

Diese Art von Chat und Datenaustausch ist gerade in kleineren Netzwerken beliebt z.B. bei Lan-Partys oder Wlan-Funknetzwerken http://Freifunk.net, da der Hub schnell eingerichtet ist und man unabhängig vom Betriebssystem Chatten und Sharen kann.

Installation¶

Open DC Hub kann direkt aus den Ubuntu-Quellen installiert werden. Dazu ist lediglich das Paket

  • opendchub (universe [2]) )

mit apturl

Paketliste zum Kopieren: 

sudo apt-get install opendchub 

sudo aptitude install opendchub

zu installieren Durch Starten des Hubs wird die Config-Datei automatisch geschrieben. Um die Konfiguration zu ändern ist es am einfachsten den Hub zu stoppen, die Konfiguration zu ändern und dann neu zu starten. Mehr Informationen dazu wie Dienste gestartet werden finden sich im entsprechenden Artikel. Hinweis: Für Precise ist kein Paket in den Quellen vorhanden.

Einrichtung¶

Benutzer anlegen¶

Es ist nicht anzuraten ein P2P-Filesharingprogramm mit Root-Rechten zu betreiben. Da der Dienst im Laufe des Installationsprozesses automatisch gestartet wird sollte er zunächst einmal gestoppt werden.

sudo /etc/init.d/opendchub stop

Solch ein Prozess sollte nie Systemdaten manipulieren dürfen. Daher sollte man für Open DC Hub einen eigenen Benutzer mit eigenem Benutzerverzeichnis anlegen [3].

sudo adduser --system --home /etc/opendchub opendchub

Unter diesem Benutzer wird Open DC Hub laufen und in dem Homeverzeichnis /etc/opendchub des neuen Benutzers werden dann die Konfigurationsdateien angelegt. Dafür ist es notwendig dem Benutzer die entsprechenden Rechte an dem während der Installation bereits erstellten Verzeichnis /etc/opendchub zu geben. Dies lässt sich durch

sudo chown -R opendchub /etc/opendchub

umsetzen.

Dienst starten¶

Nun kann man den Open DC Hub starten [3]. Dabei sollte der angelegte Benutzer angegeben werden, sowie der Pfad wo die Konfigurationsdateien abgelegt werden sollen.

sudo -u opendchub opendchub -w /etc/opendchub

Beim ersten Start werden dabei auch noch ein paar kurze Fragen gestellt.

  • Netzwerk-Port angeben (Standard wäre Port 411, allerdings müsste dazu der Hub mit Root-Rechten laufen, daher kann man hier den Port frei wählen, solange höheren Port als 1024 wählt)

  • Ein Passwort für Administrationsaufgaben

  • Ein Passwort für Verbindungen zwischen Hubs

Soll der Dienst automatisch beim Booten des Rechners gestartet werden, so kann man dafür die rc.local Datei nutzen.

Konfiguration¶

Ist der Dienst gestartet, so werden nun in /etc/opendchub einige Konfigurationsdateien abgelegt. Die wichtigste Konfigurationsdatei dabei ist die /etc/opendchub/.opendchub/config. In dieser sollten noch einige Änderungen vorgenommen werden [4], um den Dienst sicher zu betreiben. Ein paar wichtige Optionen sollen hier kurz vorgestellt werden

Name des Hubs¶

Der Name des Hubs ist mit der Option

hub_name = "Beispiel Open DC Hub"

frei wählbar.

Administration nur vom Localhost¶

Die Administration des Dienstes sollte nur vom lokalen Rechner aus möglich sein. Dies sollte man über die Option

admin_localhost = 1

ändern.

Nachricht des Tages (MOTD)¶

Loggen sich Direct-Connect-Client-Programme auf dem Server ein, so bekommen sie immer eine Nachricht angezeigt. Die so genannte "message of the day", kurz "motd". Diese Nachricht lässt sich frei anpassen. Die Meldung steht im Klartext in der Datei /etc/opendchub/.opendchub/motd und kann natürlich mit einem Editor [4] verändert werden.

Administration¶

Der Open DC Hub Dienst kann über das Telnet-Protokoll verwaltet werden. Dadurch können im laufenden Betrieb z.B. Benutzer registriert werden, Benutzer vom Server entfernt werden, Skripte geladen werden usw. Ebenso ist es möglich, diese Aktionen über den Main-Chat eines Direct Connect Client Programms abzusetzen.

Telnet¶

Um sich über Telnet zu verbinden, gibt man in einem Terminal [3] den Befehl ein

telnet localhost 53696

Dies öffnet die Verbindung zum Open DC Hub, der auf dem selben Rechner laufen muss. Jegliche Art von Eingabe fängt nun mit einem "$" an, es folgt der Befehl, dann der Wert und zum Ende ein "|". Eine Eingabe sieht dann beispielsweise so

$befehl wert|

aus.

Administrations-Passwort eingeben¶

Als Erstes muss das Administrations-Passwort eingegeben werden, das am Anfang definiert wurde

$adminpass administratorpasswort|

Befehlsübersicht¶

Mit dem Kommando

$commands|

bekommt man eine Übersicht aller möglichen Befehle

Operator festlegen¶

Benutzer können als Operatoren des Dienstes definiert werden. Dazu müssen Benutzer und Passwörter hinterlegt werden. In diesem Beispiel

$addreguser otto geheim 2|

würde der benutzer "otto" mit dem Passwort "geheim" als Operator festgelegt werden.

Main-Chat¶

Wenn man als Operator am Hub registriert ist (siehe oben), kann man auch über den Chat die Befehle eintippen. Jegliche Art von Eingabe fängt mit "!" an, nun wieder der Befehl, dann der Wert und zum Ende wieder ein "|".

!Befehl Wert|

Skripte¶

Für open DC Hub gibt es eine Vielzahl Skripte die einem Admin Arbeit abnehmen sollen. Skriptsprache ist Python. Um ein Skript zu starten, legt man es in das Verzeichnis /etc/opendchub/.opendchub/scripts ab und gibt in der telnet-Administration folgendes ein:

$reloadscripts|

Mehr dazu im Forum von opendchub-scripting
