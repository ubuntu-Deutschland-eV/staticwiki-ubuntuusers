Problembeschreibung¶

Sowohl der Download der Installations-CD von Ubuntu (ISO-Datei), als auch die dazugehörige Prüfsummendatei werden in der Regel über unverschlüsselte Links angeboten (HTTP, FTP, etc.). Um Übertragungsfehler zu vermeiden ("bit flip") hilft zwar die Prüfsumme. Da aber auch diese über einen unverschlüsselten Link heruntergeladen wird, ist nicht sichergestellt, dass es sich um die originale ISO-Datei handelt.

Zu diesem Zweck bietet Ubuntu seinen offiziellen GPG-Key an. Auch dieser wird über einen unverschlüsselten Link heruntergeladen, kann aber anhand seines Fingerprints über eine HTTPS-verschlüsselte Webseite geprüft werden.

Die Kette zum Überprüfen der ISO-Datei sieht wie folgt aus:

Fingerprint des GPG-Keys (auf verschlüsselter Webseite) -> GPG-Key -> Signatur der Prüfsummendatei -> Prüfsummendatei -> ISO-Datei

Anleitung¶

1. Download des GPG-Keys 

$ gpg --keyserver hkp://keyserver.ubuntu.com --recv-keys "8439 38DF 228D 22F7 B374 2BC0 D94A A3F0 EFE2 1092" "C598 6B4F 1257 FFA8 6632 CBA7 4618 1433 FBB7 5451"

Der Befehl lädt zwei GPG-Keys vom Ubuntu-Keyserver herunter. Keyserver verwenden das HKP-Protokoll (TCP-Port 11371). Sollte der Befehl nicht funktionieren, ist evtl. dieser Port nicht auf der Firewall freigeschaltet.

Ist die Internet-Verbindung nur über einen Proxy möglich, muss dieser als Option mitgegeben werden: 

gpg --keyserver hkp://keyserver.ubuntu.com --keyserver-options http-proxy=http://proxy.example.com:8080 --recv-keys "8439 38DF 228D 22F7 B374 2BC0 D94A A3F0 EFE2 1092" "C598 6B4F 1257 FFA8 6632 CBA7 4618 1433 FBB7 5451"

2. Auflisten der GPK-Keys (inkl. Fingerprints) 

$ gpg -k --fingerprint
...
pub   1024D/FBB75451 2004-12-30
      Key fingerprint = C598 6B4F 1257 FFA8 6632  CBA7 4618 1433 FBB7 5451
uid                  Ubuntu CD Image Automatic Signing Key <cdimage@ubuntu.com>

pub   4096R/EFE21092 2012-05-11
      Key fingerprint = 8439 38DF 228D 22F7 B374  2BC0 D94A A3F0 EFE2 1092
uid                  Ubuntu CD Image Automatic Signing Key (2012) <cdimage@ubuntu.com>
...

3. Überprüfen der Fingerprints

Die Fingerprints aus Schritt 2 müssen mit denen auf der offiziellen Ubuntu-Webseite übereinstimmen (unterer Absatz):

https://tutorials.ubuntu.com/tutorial/tutorial-how-to-verify-ubuntu#2

Achtung!

Im Webbrowser sollte die TLS-Verschlüsselung und das verwendete Zertifikat geprüft werden.

4. Signatur der Prüfsummendatei verifizieren

Zunächst muss die Signatur der Prüfsummendatei heruntergeladen werden. Dazu hängt man an den Download-Link der Prüfsummendatei einfach noch die Dateiendung .gpg an, und lädt diesen Link auf der Kommandozeile mittels wget herunter, z.B. für Ubuntu 17.10: 

$ wget http://releases.ubuntu.com/17.10/SHA256SUMS.gpg

Verwendet man einen Proxy, muss dieser als Umgebungsvariable vor dem Aufruf von wget gesetzt werden: 

$ export http_proxy=http://proxy.example.com:8080
$ wget ...

Im aktuellen Verzeichnis sollten nun drei Dateien liegen:

  • SHA256SUMS (Prüfsummendatei)

  • SHA256SUMS.gpg (Signatur der Prüfsummendatei)

  • ubuntu-17.10-desktop-amd64.iso (ISO-Datei für Ubuntu 17.10, oder ggf. ISO-Datei einer anderen Ubuntu-Version)

Nun kann die Signatur geprüft werden: 

$ gpg --verify SHA256SUMS.gpg SHA256SUMS
gpg: Signature made Dunn 19. Okt 2017 11:15:41 CEST using DSA key ID FBB75451
gpg: Good signature from "Ubuntu CD Image Automatic Signing Key <cdimage@ubuntu.com>"
gpg: WARNING: This key is not certified with a trusted signature!
gpg:          There is no indication that the signature belongs to the owner.
Primary key fingerprint: C598 6B4F 1257 FFA8 6632  CBA7 4618 1433 FBB7 5451
gpg: Signature made Dunn 19. Okt 2017 11:15:41 CEST using RSA key ID EFE21092
gpg: Good signature from "Ubuntu CD Image Automatic Signing Key (2012) <cdimage@ubuntu.com>"
gpg: WARNING: This key is not certified with a trusted signature!
gpg:          There is no indication that the signature belongs to the owner.
Primary key fingerprint: 8439 38DF 228D 22F7 B374  2BC0 D94A A3F0 EFE2 1092

Wichtig sind hier die beiden Zeilen "... Good signature ...". GPG konnte die Signatur erfolgreich prüfen.

5. ISO-Datei verifizieren 

$ sha256sum -c SHA256SUMS
ubuntu-17.10-desktop-amd64.iso: OK
sha256sum: ubuntu-17.10-server-amd64.img: No such file or directory
ubuntu-17.10-server-amd64.img: FAILED open or read
sha256sum: ubuntu-17.10-server-amd64.iso: No such file or directory
ubuntu-17.10-server-amd64.iso: FAILED open or read
sha256sum: ubuntu-17.10-server-i386.img: No such file or directory
ubuntu-17.10-server-i386.img: FAILED open or read
sha256sum: ubuntu-17.10-server-i386.iso: No such file or directory
ubuntu-17.10-server-i386.iso: FAILED open or read
sha256sum: WARNING: 4 listed files could not be read

Der Befehl kann ein wenig dauern. Wichtig ist die Ausgabe "... OK". ISO-Dateien die nicht im aktuellen Verzeichnis liegen, erzeugen natürlich eine entsprechende Fehlermeldung ("... FAILED open or read").

Problembehebung¶

Download des GPG-Keys mit Webbrowser¶

Der Download des GPG-Keys von Ubuntu ist nicht ganz unproblematisch. GPG verwendet dazu das HKP-Protokoll des Keyservers über TCP-Port 11371. Ist dieser Port auf einer Firewall nicht freigeschaltet, funktioniert der Download nicht.

Alternativ kann man aber die Keys auch über einen Webbrowser herunterladen. Dazu besucht man folgende Adresse: https://keyserver.ubuntu.com/

Im Feld "Search String:" gibt man folgendes ein:

Ubuntu CD Image Automatic Signing Key <cdimage@ubuntu.com>

und klickt auf den Button "Search!". Es wird daraufhin eine Seite mit den beiden Ubuntu-Keys angezeigt. Nach einem Klick auf die jeweilige Key-ID wird der entsprechende Key angezeigt:

-----BEGIN PGP PUBLIC KEY BLOCK-----
Version: SKS 1.1.6
Comment: Hostname: keyserver.ubuntu.com
...
-----END PGP PUBLIC KEY BLOCK-----

Bei beiden Keys muss der gesamte Block (inkl. der Zeilen "-----BEGIN ...-----" und "-----END ...-----") in eine eigene Datei kopiert werden, z.B.:

  • ubuntu.key

  • ubuntu-2012.key

Anschließend importiert man diese beiden Dateien mit GPG: 

$ gpg --import ubuntu-2012.key 
gpg: key FBB75451: public key "Ubuntu CD Image Automatic Signing Key (2012) <cdimage@ubuntu.com>" imported
gpg: Total number processed: 1
gpg:               imported: 1  (RSA: 1)
gpg: no ultimately trusted keys found

$ gpg --import ubuntu.key 
gpg: key EFE21092: public key "Ubuntu CD Image Automatic Signing Key <cdimage@ubuntu.com>" imported
gpg: Total number processed: 1
gpg:               imported: 1
gpg: no ultimately trusted keys found

Jetzt kann mit Schritt 2 fortgefahren werden.
