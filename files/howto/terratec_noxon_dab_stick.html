Terratec Noxon DAB Stick

Achtung!

Bei Problemen mit einem Howto melde dies bitte in der dazugehörigen Diskussion und wende dich bitte zusätzlich an den Verfasser des Howtos. Die Anwendung erfolgt wie üblich auf eigene Gefahr.

Hinweis:

Diese Howto-Anleitung wurde zuletzt von DPITTI am 18 Dezember 2017 unter LTS Ubuntu Mate 16.04 sowie LTS Ubuntu Mate 18.04 erfolgreich getestet. Eventuell läuft dieser Howto auch auf weiteren Linux Systemen wie Debian, Fedora, Opensuse bitte selbst testen.

Problembeschreibung¶

Dies ist eine Anleitung zur Nutzung von DAB/DAB+ Sticks zum Empfang von Digitalradio sowie Hörfunkprogrammen über (beigelegte) Antenne, Kabelanschluss oder Satellit. Es gibt zwar Linux-Treiber, jedoch keine für Linux geeignete Software als Player. Das Fraunhofer Institut lizenziert bisher keine Linuxversion des Noxon DAB Mediaplayers für Privatanwender, es wird lediglich auf eine (vermutlich kostenpflichtige) Version für den professionellen Bereich hingewiesen.

Daher wird ein angepasster Noxon DAB Mediaplayer für Windows benötigt, welcher mit Wine gestartet wird. Getestet wurden der Terratec Noxon DAB+ Stick und der Conceptronic CTVDIGUSB2 DVB-T Stick mit der jeweils beigelegten externen Antenne. Weitere Sticks mit RTL-Chip könnten auch funktionieren. Im Mysnip-Forum wird ein anderer DAB+ Player entwickelt und es werden andere Linux-taugliche Sticks vorgestellt.

Stick-Empfehlung: Der Noxon DAB+ Stick Stick funktioniert unter Linux nur in Sendernähe, da der Linux-Treiber ein sehr starkes Signal benötigt. Der Conceptronic CTVDIGUSB2 Stick ist besser und sehr empfehlenswert. Hinweis: Unter Windows haben beide Sticks besseren Empfang als unter Linux.

Anleitung¶

  1. Das System aktualisieren: 

    sudo apt-get update && sudo apt-get dist-upgrade

  2. Wine und SDR-Treiber installieren: 

     sudo apt-get install  wine playonlinux librtlsdr-dev librtlsdr0 rtl-sdr gqrx-sdr

  3. Den Noxon Player herunterladen https://www.noxonradio.ch/download/NOXON_DAB_Stick/Updates/DABPlayer5.01.zip entpacken und mit Wine oder Playonlinux installieren (Doppelklick)

  4. Danach muss eine DLL-Datei ausgetauscht und eine REG-Datei mit Registry-Einträgen per Doppelklick importiert werden. Diese sind im folgendem Archiv enthalten, welches man üblicherweise per Doppelklick öffnen und entpacken kann: http://steve-m.de/projects/rtl-sdr/rtltcpaccess.tar.gz Ausserdem kann das Skript für den Noxon Media Player auch über Playonlinux Werkzeuge Lokales Skript nur DLL Datei installiert werden https://github.com/steve-m/rtltcpaccess/archive/master.zip Oder den Skript von hier im Terminal ausführen https://github.com/steve-m/rtltcpaccess

  5. Da der Noxon Stick "nach Hause telefoniert", wird die Datei /etc/hosts mit einem Editor mit Root-Rechten bearbeitet und der Eintrag data.terratec.net am Ende der Zeile 127.0.0.1 sowie ::1 hinzugefügt, um dies zu unterbinden: 

    127.0.0.1	localhost data.terratec.net

::1 ip6-localhost ip6-loopback data.terratec.net

    Diese Datei kann man unter jedem offiziellen Ubuntu-Derivat beispielsweise so mit Root-Rechten bearbeiten: 

    sudo nano /etc/hosts

    Nach der Änderung speichern, bestätigen und schließen durch die Tasten F3, Enter sowie F2.

  6. Treiber für DVB-T zeitweise entladen, um DAB+ benutzen zu können: 

    sudo modprobe -r dvb_usb_rtl28xxu

    Hinweis:

    DVB-T kann nach Neustart bzw. neu Laden des DVB-T-Treibers wieder benutzt werden: 

    sudo modprobe dvb_usb_rtl28xxu

  7. USB-Gerätenummer (Device) ermitteln: Die Ziffer hängt vom Gerät ab und wird per lsusb ermittelt: 

    lsusb 

    Bus 001 Device 003: ID 1d19:1101 Dexatek Technology Ltd. DK DVB-T Dongle

    Bei diesem Conceptronic CTVDIGUSB2 DVB-T/ DAB Stick mit der Kennung Dexatek ist an diesem USB-Anschluss die 3 einzutragen.

  8. rtl_tcp zur Bereitstellung des DAB-Signals an beliebige Anwendungen wie den Noxon Mediaplayer in einem zweiten Termin ausführen und laufen lassen: 

    rtl_tcp -g 3

    Die 3 variiert, bitte die im vorherigen Schritt ermittelte Ziffer einsetzen. Hinweis: Die Nummer variiert, wenn man den Stick an einem anderen USB-Anschluss ansteckt. Außerdem kann dieser Befehl als Programmstarter angelegt werden, indem man als Befehl xterm -e rtl_tcp -g 3 einträgt.

  9. Noxon Mediaplayer starten: Start Icon mit folgenden Befehl ergänzen: 

     /usr/bin/q4wine

    Rechte Maustaste klicken Ausführen Durchsuche C wie folgt:

    c/Program Files/NOXON Media/NOXON DAB MediaPlayer/NOXON DAB MediaPlayer.exe

    Den zweiten Punkt kann man auch nur mit Wine machen. Weil die Noxon Player Start Exe kopiert werden muss. Das muss nur einmal gemacht werden. Ab da an Startet man den Dab Player mit 3 Klicks.

Bei den neuen Dab Player 5.01 kann man das Icon benutzen als Starter. Dazu einfach im Laufwerk C den Ordner öffnen vom Noxon Player. Dann die Dab Player exe als Starter festlegen. Wer das Erfolgreich gemacht hat startet ab jetzt mit 1 Klick.

Bei abweichenden Verzeichnisnamen kann man sich auch in einem Dateimanager durch das versteckte Verzeichnis durchklicken.

Wenn alle Schritte erfolgreich durchlaufen wurden, können mit dem soeben gestarteten Programm über den DAB/DAB+ Stick nun digitale Radiosender ausgewählt und gehört werden. Bei jedem Programmstart müssen die Schritte 8 und 9 ausgeführt oder ein passender Programmstarter erstellt werden.

Problemlösungen¶

  • Manchmal ist es notwendig, das Paket linux-firmware-nonfree zu installieren, da nicht für jeden Stick ein Treiber im Kernel enthalten ist: 

    sudo apt-get install linux-firmware-nonfree

  • Mögliche Lösungen für Fehlermeldungen, etwa beim Neuladen des Treibers, sind im Forum zu ergründen.

Bei allen Versionen von Ubuntu-Mate muss das Paket xterm installiert werden um rtl_tcp zu starten. 

 sudo apt-get install xterm

  • Das Paket q4wine kann in einigen Fällen bei Problemen mit den Noxon Media Player hilfreich sein. Zum Beispiel wenn Wine oder Playonlinux den Ordner vom Noxon Player nicht öffnen um die DLL zu tauschen.

sudo apt-get install q4wine

Hinweis:

Wer mit diesen Howto auch die Sliedeshow sehen möchte da Empfehle ich sich noch das Wine 1.8-i386 PPA: http://ubuntuhandbook.org/index.php/2015/12/install-wine-1-8-stable-new-ppa/ zu installieren. Achtung Fremdquelle!!! PPA nur zum installieren nutzen dann kann es wieder aus den Quellen entfernt werden.

Noch ein Hinweis zum Dab+ Player von Andreas Gsinn aus dem Ukw Tv Forum. Wer diese Software nutzt muss folgendes beachten wenn der Noxon Media Player installiert ist. Da der Dab+ Player von Andreas Gsinn auch ein Treiber benötigt stören sich die Programme und es kann vorkommenn das dann beide Players nicht laufen. Hier die Lösung. Den Dab+ Player und den Treiber von Andreas Gsinn mit Hilfe von PlayonLinux installieren. Dann den Noxon Media Player ohne Treiber nur mit Wine installieren. Dann laufen beide Programme habe es getestet.

Außerdem gibt es mit Dablin und Welle io zwei Native Lösung ohne Wine für Linux die ich Persönlich noch kenne. Es gibt allerdings noch mehr Software Lösungen für Dab+ und Linux deren Links ich nicht mehr weiß.
