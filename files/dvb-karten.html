DVB-Karten

Linux unterstützt eine Vielzahl von TV-Karten. In diesem Artikel geht es speziell um Hardware zum Empfang von digitalem Fernsehen (DVB). Einige Karten werden automatisch erkannt und können so direkt verwendet werden. Andere Karten erfordern dagegen zur korrekten Funktion die Installation einer hardware-spezifischen Firmware, ohne die kein Empfang möglich ist.

Hinweis:

Um eine einwandfreie Funktion zu gewährleisten, immer das Originalzubehör (Kabel) nutzen.

Ende März 2017 erfolgt in vielen Regionen Deutschlands die Umstellung auf DVB-T2 HD, die bisherige DVB-T-Ausstrahlung wurde eingestellt. Eine Abwärtskompatibilität zu DVB-T ist für den Nachfolger DVB-T2 nicht gegeben, da die Datenübertragungsverfahren auf der Funkschnittstelle inkompatibel zueinander sind. Ein reines DVB-T-Gerät kann keine DVB-T2-Signale verarbeiten.

Achtung: Ein DVB-T2-Logo am Gerät garantiert den DVB-T2-HD-Empfang noch lange nicht. Das Übertragungsverfahren wurde außerhalb Deutschlands auch ohne H265-Kodierung eingesetzt. Nur bei Fernsehern und Receivern mit dem grünen DVB-T2-HD-Logo ist der Empfang des neuen digitalen Antennenfernsehens (theoretisch) garantiert.

Erkennung¶

Automatisch¶

Wenn die Karte vom System automatisch erkannt wird, werden die notwendigen Treiber geladen und die entsprechenden Gerätedateien im Verzeichnis /dev/dvb/adapter0 angelegt:

  • demux0

  • dvr0

  • frontend0

  • net0

Das erfolgreiche Laden kann in einem Terminal [1] mit

dmesg | grep DVB

geprüft werden. Die Ausgabe sieht bei Erfolg so aus:

DVB: registering new adapter (TerraTec/qanu USB2.0 Highspeed DVB-T Receiver).
usbcore: registered new driver cinergyT2

Sollte die Karte nicht automatisch erkannt werden, hilft evtl. die Installation einer neueren Version von v4l-dvb. Bevor man diesen Weg beschreitet, sollte man aber zuerst diesen Artikel zu Ende lesen, da gegebenenfalls nur eine Firmware-Datei fehlt.

PCI-Karte¶

Nutzer einer PCI-Karte erhalten Informationen zur Karte unter anderem im Terminal [1]:

lspci

Die Ausgabe könnte so aussehen:

Multimedia controller: Philips Semiconductors SAA7146 (rev 01)

Verwendeter Chipsatz ist in diesem Fall ein Philips SAA7146.

USB-Gerät¶

Nutzer einer USB-Box oder eines USB-Sticks nutzen zum Abrufen der Informationen im Terminal [1]:

lsusb

Bei Erfolg wird z.B. dies angezeigt:

Bus 004 Device 004: ID 0ccd:0038 TerraTec Electronic GmbH Cinergy T² DVB-T Receiver

Firmware¶

Sofern die Karte nicht sofort vom System erkannt wird, kontrolliert man in einem Terminalfenster [1] das Systemprotokoll:

tail -f /var/log/messages

Beispiel:

Jun 9 18:41:05 localhost kernel: [4294818.024000] tda1004x: found firmware revision 0 -- invalid
Jun 9 18:41:05 localhost kernel: [4294818.026000] tda1004x: waiting for firmware upload...
Jun 9 18:41:05 localhost kernel: [4294818.069000] tda1004x: firmware upload failed

In diesem Beispiel konnte die Firmware nicht vom Kernelmodul tda1004x geladen werden. Um die Karte nutzen zu können, benötigt man diese aber zwingend.

Firmware¶

Die Firmware könnte in einem der folgenden zwei Pakete verfügbar sein. Am Inhalt kann man teilweise direkt erkennen, ob die benötigte Firmware enthalten ist.

  • linux-firmware (in der Standardinstallation enthalten, Inhalt Ubuntu 14.04)

mit apturl

Paketliste zum Kopieren: 

sudo apt-get install linux-firmware 

sudo aptitude install linux-firmware

  • linux-firmware-nonfree (multiverse, speziell für DVB-Karten, Inhalt Ubuntu 14.04)

mit apturl

Paketliste zum Kopieren: 

sudo apt-get install linux-firmware-nonfree 

sudo aptitude install linux-firmware-nonfree

Internet¶

Ist die Firmware in keinem Paket enthalten, muss man sie aus dem Internet herunterladen.

Installation¶

Hat man die Firmware aus dem Internet herunter geladen, muss man sie noch an den richtigen Ort im System verschieben, damit der Treiber sie auch findet.

Eine gute Übersicht findet sich unter Karten. Dort ist oftmals auch ein Download-Link für die jeweils erforderliche Firmware eingetragen.

cd /ORDNER/MIT/FIRMWARE-DATEI
sudo chown root:root FIRMWARENAME.fw           ## Eigentümer und Gruppe auf `root` ändern 
sudo mv FIRMWARENAME.fw /lib/firmware          ## verschiebt die Firmware

Anschließend sollte man das System neu starten oder das entsprechende Modul neu laden[2]. Die Firmware wird nun automatisch verwendet. Mittels [1]:

dmesg | grep MODULNAME

erhält man nun die Information, ob die Firmware gefunden wurde:

Jun 9 19:59:09 localhost kernel: [4299502.744000] tda1004x: found firmware revision 20 -- ok

TV-Empfang¶

Um DVB empfangen zu können, gibt es einige Programme, welche man dazu nutzen kann. Eine umfangreiche Liste der zur Verfügung stehenden Programme und Zusatzprogramme kann unter TV eingesehen werden. Für die Grundkonfiguration (Sendersuche) ist unter Umständen die Verwendung von dvb-apps oder alternativ w scan erforderlich.

Fernbedienung¶

Um mitgelieferte Fernbedienungen nutzen zu können, muss meist Lirc installiert und konfiguriert werden.
